## 4. 実験

### 4.1 目的
- 本研究で提案した **（例: 円柱プローブ方式）** の有効性を検証する。
- **少ない入力で安定した推定が可能か**を評価する。

### 4.2 実験環境
- OS: Ubuntu xx.xx
- CPU/GPU:
- 使用ライブラリ: Python, Open3D
- 実装バージョン: ブランチ / コミット / タグ

### 4.3 データセット
- 取得環境: （大学構内 / 実験室 / 工場想定）
- 対象物: 柱（円柱）、床（平面）など
- 点群数 / 点密度:
- センサ・SLAMの情報（あれば）

### 4.4 比較手法
- 方法A: 通常RANSAC円柱
- 方法B: Probe方式円柱
- （必要なら）直径/高さ入力あり vs なし
- （必要なら）地面制約あり vs なし

### 4.5 評価指標
- 成功率: 抽出成功の割合
- 半径誤差: |推定半径 − 実測半径|
- 長さ誤差: |推定長さ − 実測長さ|
- 軸傾き誤差: 推定軸と鉛直方向の角度差
- 操作コスト: クリック回数 / 実行時間

### 4.6 実験手順
1. 点群を読み込む
2. seed/ROI を指定
3. 各手法で推定を実行
4. 結果を保存（JSON, log）
5. 指標を算出

### 4.7 結果
- 表（成功率 / 誤差）
- 図（推定結果の可視化）

### 4.8 考察
- Probe方式が有効だった要因
- 失敗ケースの傾向
- 今後の改善点
